นโยบายความเป็นส่วนตัว — SE-Billing Extension

มีผลบังคับใช้: 11 พฤษภาคม 2026 (Effective: 2026-05-11)
ผู้พัฒนา: SE-Billing · ติดต่อ: sesurvey.ai@gmail.com

1. ภาพรวม

SE-Billing เป็น Chrome Extension (Manifest V3) ที่ช่วยกรอกค่าบริการในระบบสำรวจประกันภัย cloud.isurvey.mobi โดยอัตโนมัติตามจังหวัด/อำเภอ/ประเภทเคลม และบันทึก log การกรอกฟอร์มไปยัง backend server ที่ผู้ใช้กำหนด (default: billing.sesurvey.cloud)

2. ข้อมูลที่ extension เก็บ

2.1 ข้อมูลที่เก็บภายในเบราว์เซอร์ (chrome.storage.local)

  • Server URL — ที่อยู่ของ backend ที่ผู้ใช้กำหนดในหน้า Options
  • API Token — Bearer token สำหรับยืนยันตัวตนกับ backend

ข้อมูล 2 รายการนี้เก็บ เฉพาะในเบราว์เซอร์ของผู้ใช้ ไม่ส่งให้บุคคลที่สามใดๆ และไม่ sync ขึ้นบัญชี Google (ใช้ chrome.storage.local ไม่ใช่ sync)

2.2 ข้อมูลที่ส่งไปยัง backend ของผู้ใช้

เมื่อผู้ใช้กดปุ่ม "ยืนยันการตรวจสอบ" บนหน้าฟอร์ม cloud.isurvey.mobi extension จะถ่าย snapshot ของข้อมูลในฟอร์มที่ผู้ใช้กรอก แล้ว POST ไปที่ /api/captures บน backend ที่ผู้ใช้กำหนด ข้อมูลใน snapshot ประกอบด้วย:

  • วันที่/เวลาที่กดบันทึก (timestamp)
  • รหัสและชื่อจังหวัด / อำเภอ / ตำบล ที่เลือก
  • รหัสประเภทเคลม (MtypeID)
  • ชื่อพนักงานสำรวจ (Surveyor) ที่ปรากฏในฟอร์ม + flag ว่าเป็น SE หรือไม่
  • ชื่อผู้ตรวจสอบ (Inspector) ที่ login
  • ค่าใช้จ่าย: ค่าบริการ (SUR_INVEST), ค่าบริษัท (INS_INVEST), ค่าพาหนะ (INS_TRANS), ค่ารูป (INS_PHOTO)
  • flag/จำนวน "นอกพื้นที่" (out-of-area), "นอกเวลา" (out-of-hours)
  • จำนวนเงินหัก (deduct) และเหตุผล (ส่งช้า / เอกสารไม่ครบ)
  • โหมดที่ใช้กรอก (multi-field หรือ simple)

ข้อมูลทั้งหมดนี้คือข้อมูล ที่ผู้ใช้กรอกเองในฟอร์มของ iSurvey — extension เพียง "อ่านค่า" ที่ผู้ใช้กรอกแล้วส่งสำเนาไปเก็บที่ backend ของผู้ใช้เอง

2.3 ข้อมูลที่ extension ไม่ เก็บ

  • รหัสผ่าน, session cookie ของ iSurvey
  • ข้อมูลส่วนตัวของลูกค้าผู้เอาประกัน (ชื่อ-นามสกุล, เลขทะเบียนรถ, เลขกรมธรรม์, เลขบัตรประชาชน)
  • ประวัติการเข้าชมเว็บอื่น (browsing history)
  • ข้อมูลทางการเงิน (เลขบัตรเครดิต ฯลฯ)
  • ตำแหน่งทางภูมิศาสตร์ (GPS, IP geolocation)
  • ข้อมูลใดๆ จากเว็บไซต์อื่นนอกเหนือจาก cloud.isurvey.mobi

Privacy Policy — SE-Billing Extension

Effective date: May 11, 2026
Developer: SE-Billing · Contact: sesurvey.ai@gmail.com

1. Overview

SE-Billing is a Chrome Extension (Manifest V3) that auto-fills service fees on the insurance-survey website cloud.isurvey.mobi based on province, district, and claim type, and logs submitted forms to a backend server configured by the user (default: billing.sesurvey.cloud).

2. Data the extension handles

2.1 Stored locally in the browser (chrome.storage.local)

  • Server URL — the user-configured backend address (set in the Options page)
  • API Token — Bearer token used to authenticate with that backend

These two values are stored only in the user's browser. They are never transmitted to any third party and are not synced to the user's Google account (we use chrome.storage.local, not sync).

2.2 Data sent to the user's backend

When the user clicks the "ยืนยันการตรวจสอบ" (Confirm) button on cloud.isurvey.mobi, the extension takes a snapshot of the form values the user has entered and POSTs it to /api/captures on the user-configured backend. The snapshot contains:

  • Timestamp of the submission
  • Selected province / district (amphur) / sub-district (tumbon) — IDs and names
  • Claim type code (MtypeID)
  • Surveyor name as shown in the form, plus a flag indicating whether they are an SE surveyor
  • Inspector name as shown in the form
  • Fees: service fee (SUR_INVEST), company fees (INS_INVEST), transport (INS_TRANS), photo fee (INS_PHOTO)
  • Out-of-area / out-of-hours flags and amounts
  • Deduction amount and reason (late submission / incomplete documents)
  • Fill mode used (multi-field or simple)

All of these values are data the user themselves entered into the iSurvey form. The extension only reads them and forwards a copy to the user's own backend for record-keeping.

2.3 Data the extension does not collect

  • Passwords or session cookies for iSurvey
  • Insured customers' personal data (full names, vehicle registration, policy numbers, national ID, etc.)
  • Browsing history on other sites
  • Financial data (e.g., credit card numbers)
  • Geolocation (GPS or IP-based)
  • Any data from websites other than cloud.isurvey.mobi

3. Third-party data sharing

Limited Use Disclosure: SE-Billing's use of information received from Google APIs and from the user's browser will adhere to a Limited Use policy: the data is used solely to provide the extension's user-facing features (auto-fill and audit log) as described in this policy. We do not sell user data, do not use it for advertising, do not use it to train AI/ML models, and do not transfer it to any third party other than the user's own configured backend.

All requests are sent only to the backend the user has configured in the Options page (default: https://billing.sesurvey.cloud, the developer's server — users may replace it with their own self-hosted server at any time).

No data is sent to Google Analytics, advertising networks, trackers, or any third-party service.

4. Purpose of data collection

  • Auto-fill the iSurvey form's fee fields based on province, district, and claim type
  • Maintain an audit log of submitted forms so the user can reconcile and review billed entries
  • Export billing data to Excel for invoice preparation

5. Requested permissions and justification

  • storage — to persist Server URL and API Token in the user's browser so the extension remembers settings.
  • host_permissions: https://cloud.isurvey.mobi/* — to read form field values and auto-fill fees on the iSurvey site. The extension does not read or inject scripts on any other website.
  • host_permissions: https://billing.sesurvey.cloud/* — to fetch fee configuration and POST capture logs from the background service worker to the user-configured backend (default URL — configurable in the Options page).

6. Retention and deletion

  • Local data: Server URL and API Token are stored in chrome.storage.local. Uninstalling the extension via chrome://extensions removes them automatically.
  • Server-side capture log: stored in the SE-Billing server's SQLite database. Retention is under the user/admin's control for each instance — delete via the /admin/captures page or the API (DELETE /api/captures/:id for one record, DELETE /api/captures for all).
  • For additional server-side deletion requests, contact sesurvey.ai@gmail.com.

7. Security

  • All transport is over HTTPS (TLS 1.2+)
  • The API is protected by a Bearer token that the user configures
  • The API Token is stored in chrome.storage.local, accessible only to this extension
  • The developer's server enforces a CORS allowlist to restrict accepted origins

8. User rights

Users may:

  • Request access to their stored data
  • Request erasure of their data (right to be forgotten)
  • Uninstall the extension at any time — local storage will be cleared automatically

Exercise these rights by emailing sesurvey.ai@gmail.com.

9. Children's privacy

This extension is designed for commercial use by professional insurance surveyors — it is not directed at, and does not knowingly collect data from, children under 13.
